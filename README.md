# _SovaCE_ - Бекенд пет-проект, по создание криптобиржи

## Общая схема сервисов и как они устроены
| Сервис               | Язык             | Развёртывание     | Архитектура        | API              | Стиль / Зачем так?                              | БД                 | Брокер   |
|:---------------------|:-----------------|:------------------|:-------------------|:-----------------|:------------------------------------------------|:-------------------|:---------|
| Auth Service         | Python (FastAPI) | Микросервис       | Чистая архитектура | REST + OAuth     | Микросервис / Стандарт безопасности             | PostgreSQL + Redis | RabbitMQ |
| Trading Service      | Python (FastAPI) | Микросервис       | DDD + Onion        | REST + WebSocket | Микросервис / Изучение сложной логики (DDD)     | PostgreSQL         | Kafka    |
| Wallet Service       | Python (FastAPI) | Микросервис       | Чистая архитектура | REST + gRPC      | Микросервис / Транзакции, но проще чем трейдинг | PostgreSQL         | Kafka    |
| Notification Service | Golang           | Микросервис       | Чистая архитектура | gRPC             | Микросервис / Изучение Go, простая логика       | SQLite             | RabbitMQ |
| Analytics Service    | Python (FastAPI) | Модульный монолит | DDD                | GraphQL          | Микросервис / GraphQL + Elasticsearch           | Elasticsearch      | Kafka    |
| Admin Panel          | Python (FastAPI) | Монолит           | Onion Архитектура  | Cookie Auth      | Монолит / Изучение монолитной сборки            | PostgreSQL         | —        |


#### Объяснения почему выбрано именно эта технология, а не другая:
1. `Auth Service`
   - Способ автор. и аутентиф.
     - _Sessions_ вместо _Token-based(`JWT`)_ - потому что в криптобирже важно иметь контроль 
     над каждым пользователем, в случае мошеннических действий юзера,
     закрывать его сессию.

а потом начал переделывать_**
